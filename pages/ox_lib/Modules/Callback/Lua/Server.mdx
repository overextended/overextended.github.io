# サーバー

## クライアントコールバックのトリガー

### lib.callback

応答は別のコルーチンで処理されます。

```lua
lib.callback(name, playerId, cb, ...)
```

- name: `string`
- playerId: `number`
- cb: `function`
- ...: `any`

```lua
lib.callback('ox:getNearbyVehicles', source, function(vehicles)
    for i = 1, #vehicles do
        DeleteEntity(entity)
    end
end, args.radius)
```

### lib.callback.await

応答が受信されるまで、現在のコルーチンは中断されます。

```lua
lib.callback.await(name, playerId, ...)
```

- name: `string`
- playerId: `number`
- ...: `any`

```lua
local vehicles = lib.callback.await('ox:getNearbyVehicles', source, args.radius)

for i = 1, #vehicles do
    DeleteEntity(entity)
end
```

## サーバーコールバックの登録

### lib.callback.register

クライアントリクエストに応答するためのイベントハンドラーを登録します。

```lua
lib.callback.register(name, cb)
```

- name: `string`
- cb: `function`

```lua
lib.callback.register('ox_inventory:getItemCount', function(source, item, metadata, target)
    local inventory = target and Inventory(target) or Inventory(source)
    return (inventory and Inventory.GetItem(inventory, item, metadata, true)) or 0
end)
```
