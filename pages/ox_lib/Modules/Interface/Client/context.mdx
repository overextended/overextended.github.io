# コンテキストメニュー

## lib.registerContext

コンテキストメニューの登録に使用されます。

<Tabs items={["Lua", "JS"]}>
  <Tab>
    ```lua
    lib.registerContext(context)
    ```
  </Tab>
  <Tab>
    ```ts
    import lib from '@overextended/ox_lib/client';

    lib.registerContext(context);
    ```

  </Tab>
</Tabs>

- id: `string`
  - 一意のメニュー識別子。メニューを開くために使用されます。
- title: `string`
  - メニューに表示されるタイトル。マークダウンをサポートしています。
- menu?: `string`
  - メニュー識別子 - 定義されている場合、メニュータイトルの横に定義したメニューに移動する戻る矢印が表示されます。
- canClose: `boolean`
  - falseに設定すると、ユーザーはいずれかのボタンを押さずにメニューを終了できなくなります。
- onExit?: `function`
  - ユーザーがESCでコンテキストメニューを閉じたときに実行される関数。
- onBack?: `function`
  - ユーザーが前のメニューに戻るために戻るボタンを押したときに実行される関数。
- options: `table` (`object` または `array`)
  - item: `key` (`string`) または `table` (`object`)
    - title?: `string`
      - キーを使用しない場合は、ボタンのタイトルを設定します。マークダウンをサポートしています。
    - disabled?: `boolean`
      - ボタンをグレー表示し、クリックできないようにします。
    - readOnly? `boolean`
      - すべてのホバーおよびアクティブスタイルを削除し、定義されている場合はonSelectを無効にします。
    - menu?: `string`
      - ボタンが移動するメニュー識別子。定義されている場合は矢印。
    - onSelect: `function`
      - ボタンがクリックされたときに実行される関数。
    - icon?: `string`
      - 左側に表示されるFontAwesomeアイコン。通知およびtextuiアイコンと同じように機能します。
      - 画像URL、pngおよびwebpファイルもサポートしていますが、FontAwesomeアイコンよりも使用することはお勧めしません。
    - iconColor?: `string`
      - 表示されるアイコンの色。
    - iconAnimation?: `'spin'` `'spinPulse'` `'spinReverse'` `'pulse'` `'beat'` `'fade'` `'beatFade'` `'bounce'` `'shake'`
    - progress?: `number`
      - このパーセンテージまで満たされたプログレスバーを追加します
    - colorScheme?: `string`
      - プログレスバーの配色を設定します。現在のオプションは次の場所にあります。
        - https://v6.mantine.dev/theming/colors/#default-colors
        - 例：`blue` または `teal`
    - arrow?: `boolean`
      - イベントからメニューを開くときに便利な、`menu` と同様に右側に矢印を表示します。falseに設定すると非表示にできます。
    - description?: `string`
      - キーとして定義されているボタンタイトルの下に表示される説明。マークダウンをサポートしています。
    - image?: `string`
      - ボタンのメタデータに表示される画像へのURL。
    - metadata?: `string[]` または `object` または `array`
      - ボタンにカーソルを合わせると横に表示される情報。
      - label: `string`
      - value: `any`
      - progress?: `number`
        - メタデータにプログレスバーを表示します。
      - colorScheme?: `string`
        - 上記と同じ。
    - event?: `string`
      - ボタンがトリガーするイベント。
    - serverEvent?: `string`
      - ボタンがトリガーするサーバーイベント。
    - args?: `any`
      - イベントまたはonSelect関数に送信される引数。

1つの `lib.registerContext` 関数で好きなだけコンテキストメニューを登録できます。

メニューは、記述した順序にするか、アルファベット順にソートすることができます。
メニューをアルファベット順にソートするには、ボタン（および/またはメタデータ）をキーとして定義する必要があります。それ以外の場合、キーを使用せず、代わりにテーブルを使用すると、メニューは定義した順序になります。

## lib.showContext

登録されたコンテキストメニューをIDで開きます。

<Tabs items={["Lua", "JS"]}>
  <Tab>
    ```lua
    lib.showContext(id)
    ```
  </Tab>
  <Tab>
    ```ts
    import lib from '@overextended/ox_lib/client';

    lib.showContext(id);
    ```

  </Tab>
</Tabs>

- id: `string`

## lib.hideContext

現在表示されているコンテキストメニューを非表示にします。

<Tabs items={["Lua", "JS"]}>
  <Tab>
    ```lua
    lib.hideContext(onExit)
    ```
  </Tab>
  <Tab>
    ```ts
    import lib from '@overextended/ox_lib/client';

    lib.hideContext(onExit);
    ```

  </Tab>
</Tabs>

- onExit: `boolean`
  - メニューのonExit関数を実行するかどうかを定義します。

## lib.getOpenContextMenu

現在開いているコンテキストメニューのIDを返します。

コンテキストメニューが開いていない場合は `nil` を返します。

<Tabs items={["Lua", "JS"]}>
  <Tab>
    ```lua
    lib.getOpenContextMenu()
    ```
  </Tab>
  <Tab>
    ```ts
    import lib from '@overextended/ox_lib/client';

    lib.getOpenContextMenu();
    ```

  </Tab>
</Tabs>

## 使用例

まず、指定されたオプションでメニューを登録し、次にコマンドでshow関数を呼び出します。

<Callout>
  外部の値に依存しないメニュー（別名静的メニュー）を常に再登録しないでください。
</Callout>

<Tabs items={["Lua", "JS"]}>
  <Tab>
    ```lua
    lib.registerContext({
      id = 'some_menu',
      title = 'いくつかのコンテキストメニュー',
      options = {
        {
          title = '空のボタン',
        },
        {
          title = '無効なボタン',
          description = 'このボタンは無効です',
          icon = 'hand',
          disabled = true
        },
        {
          title = 'サンプルボタン',
          description = 'サンプルボタンの説明',
          icon = 'circle',
          onSelect = function()
            print("ボタンを押しました！")
          end,
          metadata = {
            {label = '値1', value = 'いくつかの値'},
            {label = '値2', value = 300}
          },
        },
        {
          title = 'メニューボタン',
          description = '別のメニューに移動します！',
          menu = 'other_menu',
          icon = 'bars'
        },
        {
          title = 'イベントボタン',
          description = 'イベントからメニューを開き、イベントデータを送信します',
          icon = 'check',
          event = 'test_event',
          arrow = true,
          args = {
            someValue = 500
          }
        }
      }
    })
    ```
  </Tab>
  <Tab>
    ```ts
    import lib from '@overextended/ox_lib/client';

    lib.registerContext({
      id: 'some_menu',
      title: 'いくつかのコンテキストメニュー',
      options: [
        {
          title: '空のボタン',
        },
        {
          title: '無効なボタン',
          description: 'このボタンは無効です',
          icon: 'hand',
          disabled: true,
        },
        {
          title: 'サンプルボタン',
          description: 'サンプルボタンの説明',
          icon: 'circle',
          onSelect: () => {
            console.log('ボタンを押しました！');
          },
          metadata: [
            { label: '値1', value: 'いくつかの値' },
            { label: '値2', value: 300 },
          ],
        },
        {
          title: 'メニューボタン',
          description: '別のメニューに移動します！',
          menu: 'other_menu',
          icon: 'bars',
        },
        {
          title: 'イベントボタン',
          description: 'イベントからメニューを開き、イベントデータを送信します',
          icon: 'check',
          event: 'test_event',
          arrow: true,
          args: {
            someValue: 500,
          },
        },
      ],
    });
    ```

  </Tab>
</Tabs>

次に、`other_menu` という名前の2番目のメニューも登録できます。

<Tabs items={["Lua", "JS"]}>
  <Tab>
    ```lua
    lib.registerContext({
      id = 'other_menu',
      title = 'その他のコンテキストメニュー',
      menu = 'some_menu',
      onBack = function()
        print('戻りました！')
      end,
      options = {
        {
          title = 'ここには何もありません'
        }
      }
    })
  ```
  </Tab>
  <Tab>
    ```ts
    lib.registerContext({
      id: 'other_menu',
      title: 'その他のコンテキストメニュー',
      menu: 'some_menu',
      onBack: () => {
        console.log('戻りました！');
      },
      options: [
        {
          title: 'ここには何もありません',
        },
      ],
    });
    ```
  </Tab>
</Tabs>

そして、`some_menu` メニューから実行するイベント。これにより、別のメニューが開きます。

<Tabs items={["Lua", "JS"]}>
  <Tab>
    ```lua
    RegisterNetEvent('test_event', function(args)
      lib.registerContext({
        id = 'event_menu',
        title = 'イベントメニュー',
        menu = 'some_menu',
        options = {
          {
            title = 'イベント値： '..args.someValue,
          }
        }
      })

      lib.showContext('event_menu')
    end)
    ```

  </Tab>
  <Tab>
    ```ts
    onNet('test_event', (args: { someValue: number }) => {
      lib.registerContext({
        id: 'event_menu',
        title: 'イベントメニュー',
        menu: 'some_menu',
        options: [
          {
            title: `イベント値： ${args.someValue}`,
          },
        ],
      });

      lib.showContext('event_menu');
    });
    ```

  </Tab>
</Tabs>

最後に、`some_menu` メニューを表示するためのテストコマンドを登録します。

<Tabs items={["Lua", "JS"]}>
  <Tab>
    ```lua
    RegisterCommand('testcontext', function()
      lib.showContext('some_menu')
    end)
    ```
  </Tab>
  <Tab>
    ```ts
    RegisterCommand('testcontext', () => {
      lib.showContext('some_menu');
    });
    ```
  </Tab>
</Tabs>

メニューの `args` テーブルのデータは、登録するイベントの最初の引数として渡されます。

このイベントを使用して、独自のオプションを持つ新しいコンテキストメニューも登録します。

最初のメニューのIDになるように `menu` パラメータを定義することにより、メニュータイトルの横に戻る矢印ボタンが表示され、そこから戻ることができます。
