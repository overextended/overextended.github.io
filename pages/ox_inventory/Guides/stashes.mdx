# カスタムスタッシュ

エクスポートされたRegisterStash関数を利用して、リソースの外部からカスタムスタッシュを設定できます。

まず、スタッシュのプロパティを定義する必要があります。

## スタッシュプロパティ

- id: `string`
  - データベースでスタッシュを識別するための一意の名前。
- label: `string`
  - スタッシュを表示するときの表示名。
- slots: `number`
  - スタッシュが持つスロットの数。
- weight: `number`
  - スタッシュインベントリの最大重量。
- owner?: `string` または `boolean`
  - `true`: 各プレイヤーは独自のスタッシュを持っていますが、他のプレイヤーのスタッシュを開くように要求できます
  - `false`: 単一のスタッシュのみが存在し、すべてのプレイヤー間で共有されます
  - `string`: スタッシュは明示的に指定された所有者に属し、通常はプレイヤー識別子です
- groups?: `table`
  - スタッシュにアクセスできるジョブ名と最小グレードのキーと値のペア。（`{["police"] = 0, ["ambulance"] = 2}`）
  - name: `string`
  - grade: `number`
- coords?: `vector3` または `table`
  - プレイヤーが十分に近くにいない場合にスタッシュが開かれるのを防ぐために、スタッシュの座標を設定できます。
  - スタッシュの座標を含むベクトルまたはテーブル。

## 例

以下では値がハードコードされていますが、データベースからロードすることもできます（特に、所有者などの不明なフィールドがある場合）

```lua
-- サーバー
local stash = {
    id = '42wallabyway',
    label = '42 Wallaby Way',
    slots = 50,
    weight = 100000,
    owner = 'char1:license'
}

AddEventHandler('onServerResourceStart', function(resourceName)
    if resourceName == 'ox_inventory' or resourceName == GetCurrentResourceName() then
        exports.ox_inventory:RegisterStash(stash.id, stash.label, stash.slots, stash.weight, stash.owner)
    end
end)

-- クライアント
exports.ox_inventory:openInventory('stash', {id='42wallabyway', owner=property.owner})
```

次のサンプルは、esx_propertyのDBデータに基づいています。

```lua
-- サーバー
local properties

MySQL.query('SELECT * FROM `properties`', {}, function(result)
    properties = result
end)

RegisterNetEvent('ox:loadStashes', function(id)
local stash = properties[id]
    if stash then
        -- id: 1, name: WhispymoundDrive, label: 2677 Whispymound Drive, coords: {"x":118.748,"y":566.573,"z":175.697}
        ox_inventory:RegisterStash(stash.name, stash.label, 50, 100000, true, false, json.encode(stash.room_menu))
    end
end)

-- クライアント
local ox_inventory = exports.ox_inventory

if ox_inventory:openInventory('stash', property.id) == false then
    TriggerServerEvent('ox:loadStashes')
    ox_inventory:openInventory('stash', property.id)
end
```
