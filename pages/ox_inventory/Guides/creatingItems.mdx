# アイテムの作成

## アイテムデータの定義

ゲーム内でアイテムを表示または使用する前に、まず定義する**必要があります**。

すべてのアイテムは、キーと値のペアを持つ[/data/items.lua](https://github.com/overextended/ox_inventory/blob/main/data/items.lua)ファイルで定義されます。
キーはアイテムの名前（ラベルではない）であり、値はアイテムのオプションを含むテーブルです。

- アイテムオプション：`table`
  - label: `string`
  - weight?: `number`
  - stack?: `boolean`
    - falseに設定すると、アイテムのスタックが許可されません。
  - degrade?: `number`
    - アイテムが劣化するまでの時間（分）。
  - decay?: `boolean`
    - trueの場合、耐久性が0になるとアイテムは削除されます（劣化したアイテムの場合は即時ではありません）。
  - close?: `boolean`
    - falseに設定すると、アイテム使用時にインベントリを閉じません。
  - description?: `string`
    - ツールチップに表示されるアイテムの説明
  - consume?: `number`
    - 必要なアイテム数と削除された使用量。
    - デフォルト：1
    - 小数に設定すると、代わりに耐久性を消費します（0.2 = 20％）。
  - allowArmed?: `boolean`
    - trueに設定すると、武器を装備している間でもアイテムの使用が許可されます。
  - server?: `table`
    - export?: `string`
  - client?: `table`
    - export?: `string`
      - アイテム使用後にトリガーされるエクスポート。
    - event?: `string`
      - アイテム使用後にトリガーされるイベント。
    - status?: `table`
      - 使用後にesx_status値を調整します。
    - anim?: `table`
      - プログレスバー中に再生されるアニメーション。
      - dict: `string`
      - clip: `string`
    - prop?: `table`
      - プログレスバー中に表示される添付プロップ。
      - model: `string` または `hash`
      - pos: `table` (x, y, z)
      - rot: `table` (x, y, z)
      - bone?: `number`
      - rotOrder?: `number`
    - disable?: `table`
      - プログレスバー中に無効にするアクション。
      - move?: `boolean`
      - car?: `boolean`
      - combat?: `boolean`
      - mouse?: `boolean`
      - sprint?: `boolean`
    - usetime?: `number`
    - cancel?: `boolean`
      - trueに設定すると、プレイヤーはアイテムの使用をキャンセルできます。
    - add?: `function`(total: `number`)
      - アイテムを受け取ったときにトリガーされる関数
      - 合計アイテム数を `total` として返します
    - remove?: `function`(total: `number`)
      - アイテムを削除したときにトリガーされる関数
      - 合計アイテム数を `total` として返します
  - buttons?: `table`
    - label: `string`
    - action: `function`(slot: `number`)
      - コンテキストメニューでボタンがクリックされたときのコールバック関数。アイテムスロットを返します。

## アイテムを使用可能にする

- ESXを使用している場合は、引き続き `ESX.RegisterUsableItem` を使用できます。
- QBoxを使用している場合は、引き続き `exports.qbx_core:CreateUseableItem` を使用できます。

組み込みシステムを使用する方が安全で、はるかに多くの機能を提供します。

### クライアントコールバック

アイテムコールバックは、エクスポートを定義するか（推奨）、[items/client.lua](https://github.com/overextended/ox_inventory/blob/main/modules/items/client.lua#L33)に追加することで追加できます。

[アイテムデータ](https://github.com/overextended/ox_inventory/blob/main/data/items.lua#L11)を定義するときに、client.exportを追加すると、アイテム使用時にイベントがトリガーされます。
正しい形式は `export = resourceName.exportName` です。

```lua
exports('bandage', function(data, slot)
    local playerPed = PlayerPedId()
    local maxHealth = GetEntityMaxHealth(playerPed)
    local health = GetEntityHealth(playerPed)

    -- 歩行者は回復する必要がありますか？アイテムの使用をキャンセルできます。
    if health < maxHealth then
        -- アイテムを正しく使用するための内部コードをトリガーします。
        -- これにより、セキュリティが向上し、使用時にアイテムが削除され、プログレスバーのサポートが追加され、サーバーコールバックに必要になります。
        exports.ox_inventory:useItem(data, function(data)
            -- サーバーはアイテムが使用できることを確認しました。
            if data then
                SetEntityHealth(playerPed, math.min(maxHealth, math.floor(health + maxHealth / 16)))
                lib.notify({description = 'もう気分が良くなりました'})
            end
        end)
    else
        -- アイテムを使用しないでください
        lib.notify({type = 'error', description = '今は包帯は必要ありません'})
    end
end)
```

### サーバーコールバック

サーバーでコールバック関数を定義して、いくつかのイベント（usingItem、usedItem、buyItem）を処理できます。
これは、エクスポート（推奨）にするか、[items/server.lua](https://github.com/overextended/ox_inventory/blob/main/modules/items/server.lua)の末尾に追加できます。
[アイテムデータ](https://github.com/overextended/ox_inventory/blob/main/data/items.lua#L14)を定義するときに、server.exportを追加すると、上記のアクションのイベントがトリガーされます。
正しい形式は `export = resourceName.exportName` です。

```lua
exports('bandage', function(event, item, inventory, slot, data)
    -- プレイヤーはアイテムを使用しようとしています。
    if event == 'usingItem' then
        local playerPed = GetPlayerPed(inventory.id)
        local maxHealth = GetEntityMaxHealth(playerPed)
        local health = GetEntityHealth(playerPed)

        -- プレイヤーが回復する必要があるかどうかを確認します。
        if health >= maxHealth then
            TriggerClientEvent('ox_lib:notify', inventory.id, {type = 'error', description = '今は包帯は必要ありません'})

            -- 'false'を返すと、アイテムの使用が妨げられます
            return false
        end

        return
    end

    -- プレイヤーはアイテムの使用を終了しました。
    if event == 'usedItem' then
        return TriggerClientEvent('ox_lib:notify', inventory.id, {description = 'もう気分が良くなりました'})
    end

    -- プレイヤーはアイテムを購入しようとしています。
    if event == 'buying' then
        return TriggerClientEvent('ox_lib:notify', inventory.id, {type = 'success', description = '包帯を購入しました'})
    end
end)
```

## コンテナアイテムの作成

他のアイテムと同様に、まずアイテムを登録する必要があります。

登録したら、`/modules/items/containers.lua` でアイテムをコンテナとして定義できます。
コンテナのキーは、アイテムを登録したときに付けた `name` です。
スロット数、最大重量、ブラックリストアイテム、ホワイトリストアイテムも定義できます。

- itemName:
  - slots: number
  - 数値はスロット数を表します
  - maxWeight: number
  - 数値はコンテナ内の最大重量を表します
  - blacklist:
    - 単一および複数のアイテムをサポート
    - `{ 'testburger', 'testburger2' }`
  - whitelist:
    - 単一および複数のアイテムをサポート
    - `{ 'testburger', 'testburger2' }`

### Example

```lua filename="Register Example"
['paperbag'] = {
    label = 'Paper Bag',
    weight = 1,
    stack = false,
    close = false,
    consume = 0
},
```

```lua filename="Properties Example"
setContainerProperties('paperbag', {
	slots = 5,
	maxWeight = 1000,
	blacklist = { 'testburger' }
})
```
