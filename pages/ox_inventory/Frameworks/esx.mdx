# ESX

- [ESX Legacyの互換バージョン](https://github.com/esx-framework/esx_core)（1.6.0以降）が必要です。
- `convertinventory esx` を使用してプレイヤーインベントリを変換できます。
- データベース内のすべてのアイテムは内部アイテムデータに移行されます。
  - アイテムが移行されたら、必ずリソースを再起動してください！

## 互換性

Ox Inventoryは、既存のアイテム、インベントリ、および武器システムを完全に置き換えるものです。ESXおよびデフォルトの動作に依存するリソースとは本質的に互換性がありません。

- ロードアウトは存在せず、武器はアイテムとして扱われます。
- 標準のショップとスタッシュ（例：esx_shops、esx_weaponshop、esx_policejob）。
- デフォルトのESXインベントリを変更したり、UIを提供したりするリソース（例：esx_trunkinventory、esx_inventoryhud）。

## インストール

- [ESX Legacyの互換バージョン](https://github.com/esx-framework/esx-legacy)（1.6.0以降）を使用します。
- `server.cfg`を変更し、es_extendedの直後にox_inventoryを開始します。

```yaml
start oxmysql
start ox_lib
start es_extended
start qtarget
start ox_inventory
```

## ESXインベントリデータの変換

- サーバーを起動し、サーバーコンソールに `convertinventory esx` と入力します。
- オプションで、サーバーコンソールに `convertinventory esxproperty` と入力します。
- 変換が完了したら、サーバーを再起動します。

## オプションの最適化

`xPlayer.getInventoryItem`など、xPlayerのすべてのアイテム関連関数は互換性のために変更されていますが、非推奨と見なされます。

理由は非常に単純です。考慮すべき追加の関数参照とオーバーヘッドが発生するようになりました。幸いなことに、新しいインベントリ関数は直接使用でき、古い関数よりも大幅に改善されています。

詳細については関数セクションを読む必要がありますが、以下で十分なアイデアが得られるはずです。
