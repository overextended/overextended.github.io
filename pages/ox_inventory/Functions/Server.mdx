# サーバー

## setPlayerInventory

プレイヤーのインベントリを作成して設定します。

```lua
exports.ox_inventory:setPlayerInventory(player, data)
```

- player: `table`
  - source: `number`
  - identifier: `string`
  - name: `string`
  - groups?: `table`
  - sex?: `string`
  - dateofbirth?: `string`
- data?: `table`
  - 提供されていない場合、データベースからプレイヤーのインベントリデータをロードします。

## forceOpenInventory

渡されたデータを使用してインベントリを開きます。
通常のセキュリティチェック（グループ、座標）なしで、プレイヤーにインベントリを開くことを強制します。

```lua
exports.ox_inventory:forceOpenInventory(playerId, invType, data)
```

- playerId: `number`
- invType: `string`
  - `'player'`
  - `'stash'`
  - `'container'`
  - `'drop'`
  - `'glovebox'`
  - `'trunk'`
  - `'dumpster'`
- data: `number` または `string` または `table`

<Tabs items={["Player", "Stash"]}>
  <Tab>
    ターゲットプレイヤーのインベントリを開きます。

    ```lua
    exports.ox_inventory:forceOpenInventory(1, 'player', 3)
    ```

    プレイヤーのインベントリを開くための管理者コマンド。

    ```lua
    RegisterCommand('openplayerinv', function(source, args)
        exports.ox_inventory:forceOpenInventory(source, 'player', tonumber(args[1]))
    end, true)
    ```
  </Tab>
  <Tab>

    カスタムスタッシュを開きます（サーバーでRegisterStashを使用して作成）。

    ```lua
    exports.ox_inventory:forceOpenInventory(1, 'stash', 'society_police')
    ```

    特定の所有者のスタッシュを開きます。

    ```lua
    exports.ox_inventory:forceOpenInventory(1, 'stash', { id = 'police_locker', owner = 'license:xxxxxxxx' })
    ```
  </Tab>
</Tabs>

## Items

登録されているすべてのアイテムのテーブルを返します。形式は[data/items.lua](https://github.com/overextended/ox_inventory/blob/main/data/items.lua)で定義されているとおりです。

オプションでアイテムの名前を取り、そのアイテムのデータのみを返します（すべてのデータを取得することはお勧めしません）。

```lua
exports.ox_inventory:Items(itemName)
```

- itemName?: `string`

次のスニペットは、データベースにクエリを実行するのではなく、okokCraftingやcore_craftingなどのクラフトリソースで使用できます。

```lua
local itemNames

ESX.RegisterServerCallback('crafting:itemNames', function(source, cb)
    if not itemNames then
        itemNames = {}
        for item, data in pairs(exports.ox_inventory:Items()) do
            itemNames[item] = data.label
        end
    end
    cb(itemNames)
end)
```

## AddItem

指定されたインベントリにアイテムを追加します。

[CanCarryItem](#cancarryitem)と一緒に使用する必要があります。そうしないと、最大重量を超える可能性があります。

```lua
exports.ox_inventory:AddItem(inv, item, count, metadata, slot, cb)
```

- inv: `table` または `string` または `number`
  - インベントリの一意のID、またはIDと所有者を含むテーブル。
    - playerId: `1`
    - inventoryId: `gloveVGH283`
    - `{ id = 'personallocker', owner = 'license:xxxxxx'}`
- item: `string`
  - ターゲットに追加するアイテムの名前。
- count: `number`
  - 追加するアイテムの数。
- metadata?: `table` または `string`
  - アイテムオブジェクトに添付する一意のデータのテーブル。文字列は「type」フィールドを持つテーブルを作成します。
- slot?: `number`
  - アイテムを追加する特定のスロット。スロットが無効な場合、最初に使用可能なスロットが代わりに使用されます。
- cb?: function(success: `boolean`, response?: `string`)

グローブボックス、トランク、またはスタッシュに使用する場合、最初に[GetInventory](./GetInventory.md)でインベントリがロードされていることを確認する必要があります。

cbが未定義の場合は `success, response` を返します。それ以外の場合は、コールバックでのみ使用されます。

**失敗時の「response」引数の可能な値：**

- "invalid_item"：アイテムが存在しません
- "invalid_inventory"：インベントリが存在しません
- "inventory_full"：空きスロットがありません

<br />
<u>**Example**</u>

```lua
local success, response = exports.ox_inventory:AddItem('gloveVGH283', 'bread', 4)

if not success then
    -- 利用可能なスロットがない場合、値は「inventory_full」になります
    return print(response)
end

print(json.encode(response, {indent=true}))
--[[
    {
        "metadata": [],
        "label": "Bread",
        "slot": 1,
        "stack": true,
        "close": true,
        "name": "bread",
        "count": 1,
        "weight": 150
    }
]]
```

## RemoveItem

指定されたインベントリから指定されたアイテムを削除します。

```lua
exports.ox_inventory:RemoveItem(inv, item, count, metadata, slot, ignoreTotal)
```

- inv: `table` または `string` または `number`
  - インベントリの一意のID、またはIDと所有者を含むテーブル。
    - playerId: `1`
    - inventoryId: `gloveVGH283`
    - `{ id = 'personallocker', owner = 'license:xxxxxx'}`
- item: `string`
  - ターゲットから削除するアイテムの名前。
- count: `number`
  - 削除するアイテムの数。
- metadata?: `table` または `string`
  - 一致するメタデータプロパティを持つアイテムのみを削除します。
- slot?: `number`
  - アイテムを削除する特定のスロット。スロットが無効な場合、最初に使用可能なスロットが代わりに使用されます。
- ignoreTotal?: `boolean`
  - 指定された数まで、できるだけ多くのアイテムを削除します。

成功：`boolean`、応答：`string?` を返します。

**失敗時の「response」の可能な値：**

- "invalid_item"：アイテムが存在しません
- "invalid_inventory"：インベントリが存在しません
- "not_enough_items"：インベントリに指定されたアイテムが十分にありませんでした

<br />
<u>**Example**</u>

```lua
-- 指定されたプレートのグローブボックスから水を2つ削除します。
local success = exports.ox_inventory:RemoveItem('gloveVGH283', 'water', 2)
```

## GetItem

指定されたインベントリから_一般的な_アイテムデータを合計数とともに返します。

```lua
exports.ox_inventory:GetItem(inv, item, metadata, returnsCount)
```

- inv: `table` または `string` または `number`
- item: `table` または `string`
  - アイテム配列にすることができます。
- metadata?: `any`
  - 指定されたメタデータに厳密に一致するアイテムの数のみを返します。
- returnsCount?: `boolean`

`returnsCount` がtrueに設定されている場合、返される値はアイテムが見つかった回数に基づく `count` になります。
それ以外の場合は、アイテムに関連するデータ _と_ インベントリで見つかった合計数を返します。

<br />
<u>**Example**</u>

```lua
local item = ox_inventory:GetItem(source, 'water', nil, false)

print(json.encode(item, {indent=true}))
--[[
    {
        "consume": 1,
        "count": 15,
        "stack": true,
        "name": "water",
        "weight": 500,
        "label": "Water",
        "close": true
    }
]]
```

## CanCarryItem

インベントリが指定されたアイテムを運ぶことができるかどうかに応じてtrueまたはfalseを返します。

この関数は、インベントリの重量と利用可能なスロットを確認します。

```lua
exports.ox_inventory:CanCarryItem(inv, item, count, metadata)
```

- inv: `table` または `string` または `number`
- item `table` または `string`
  - アイテムの配列にすることができます。
- count: `number`
- metadata?: `table` または `string`
  - metadataが文字列として渡された場合、`metadata.type` がチェックされます。

<br />
<u>**Example**</u>

```lua
-- イベントを呼び出しているプレイヤーが水を3つ運ぶことができるかどうかを確認します
if exports.ox_inventory:CanCarryItem(source, 'water', 3) then
    -- 運ぶことができる場合の処理
else
    -- 運ぶことができない場合の処理
end
```

```lua
exports.ox_inventory:CanCarryAmount(inv, item)
```

- inv: `table` または `string` または `number`
- item: `table` または `string`

## CanCarryWeight

インベントリが指定された重量を運ぶことができるかどうか、および空きインベントリ重量を返します。

```lua
exports.ox_inventory:CanCarryWeight(inv, weight)
```

- inv: `table` または `string` または `number`
- weight: `number`

<br />
<u>**Example**</u>

```lua
-- プレイヤーが1000グラム運べるかどうかを確認します。
local fillAmount = 1000
local canCarryWeight, freeWeight = ox_inventory:CanCarryWeight(playerId, fillAmount)

if freeWeight == 0 then
  -- プレイヤーは重量を運ぶことができません。
  return
elseif not canCarryWeight then
  -- インベントリが指定された重量を運ぶことができないため、fillAmountを変更します
  fillAmount = freeWeight
end

-- 何かをする
```

## RegisterStash

新しいカスタムスタッシュを作成します。

```lua
exports.ox_inventory:RegisterStash(id, label, slots, maxWeight, owner, groups, coords)
```

- id: `string` または `number`
  - データベースからロードするときのスタッシュ識別子。
- label: `string`
  - インベントリが開いているときの表示名。
- slots: `number`
- maxWeight: `number`
- owner: `string` または `boolean` または `nil`
  - `string`：所有者にリンクされているスタッシュにのみアクセスできます。
  - `true`：各プレイヤーは一意のスタッシュを持っていますが、他のプレイヤーのスタッシュを要求できます。
  - `nil`：常に共有されます。
- groups: `table`
  - スタッシュにアクセスできるプレイヤーグループ（ジョブ）のテーブル。
  - 数値が最小必要グレードであるグループ名のテーブル。
  - `{['police'] = 0, ['ambulance'] = 2}`
- coords?: `vector3` または `vector3[]`

<Callout>
  プレイヤーがスタッシュを開く前に、この関数をトリガーする必要があります。
</Callout>

<br />
<u>**Example**</u><br />
この関数のユースケース例については、作成された[ガイド](../Guides/stashes)を確認してください。

## CreateTemporaryStash

しばらくすると削除される一時的なスタッシュを作成します。

```lua
exports.ox_inventory:CreateTemporaryStash(properties)
```

- properties: `table`
  - label: `string`
  - slots: `number`
  - maxWeight: `number`
  - owner?: `string` `number` または `boolean`
    - `string`：所有者にリンクされているスタッシュにのみアクセスできます。
    - `true`：各プレイヤーは一意のスタッシュを持っていますが、他のプレイヤーのスタッシュを要求できます。
    - インベントリは、`false` または `nil` の場合は常に共有されます。
  - groups?: `table<string, number>`
    - 数値が最小必要グレードであるグループ名（ジョブなど）のテーブル。
    - `{['police'] = 0, ['ambulance'] = 2}`
  - coords?: `vector3`
    - スタッシュは近くにいる場合にのみアクセスできます。
  - items?: `{ [number]: string, [number]: number, [number]?: table }[]`
    - itemName、count、metadataのシーケンスを含むテーブルの配列。

Return:
- inventoryId: `string`

<br />
<u>**Example**</u>

```lua
local mystash = exports.ox_inventory:CreateTemporaryStash({
    label = 'mystash',
    slots = 5,
    maxWeight = 5000,
    items = {
        { 'WEAPON_MINISMG', 1 },
        { 'ammo-9', 69 },
        { 'water', 2, { label = 'Mineral water' } }
    }
})

TriggerClientEvent('ox_inventory:openInventory', 1, 'stash', mystash)
```

## CustomDrop

ドロップは他のリソースから作成でき、さまざまなアイテムを含み、カスタムラベルを利用できます（「ドロップ32648」の代わりに）。

```lua

```

- prefix: `string`
- items: `table`
  - name: `string`
  - count: `number`
  - metadata?: `table`
- coords: `vector3`
- slots?: `number`
- maxWeight?: `number`
- instance?: `string` または `number`
- model?: `number`

```lua
-- マーカー付きの一般的なドロップを作成します
exports.ox_inventory:CustomDrop('Carcass', {
    {'meat', 5, { grade = 2, type = 'deer' }},
    {'hide', 5, { grade = 2, type = 'deer' }}
}, coords)

-- エンティティ付きのドロップを作成します

    { 'WEAPON_MINISMG', 1 },
    { 'ammo-9', 69 },
}, GetEntityCoords(GetPlayerPed(1)), 5, 10000, nil, `w_sb_minismg`)
```

## SetDurability

指定されたスロットに耐久性を設定します。

武器の修理に使用できます。

```lua
exports.ox_inventory:SetDurability(inv, slot, durability)
```

- inv: `table` または `string` または `number`
- slot: `number`
- durability: `number`

<br />
<u>**Example**</u>

```lua
local ox_inventory = exports.ox_inventory

-- ソースプレイヤーのインベントリのスロット3のアイテムの耐久性を100に設定します
ox_inventory:SetDurability(source, 3, 100)

-- ソースプレイヤーの現在の武器の耐久性を100に設定します
local weapon = ox_inventory:GetCurrentWeapon(source)

if weapon then
    ox_inventory:SetDurability(source, weapon.slot, 100)
end
```

## SetMetadata

指定されたスロットにメタデータを設定します。

```lua
ox_inventory:SetMetadata(inv, slot, metadata)
```

- inv: `table` または `string` または `number`
- slot: `number`
- metadata: `table`

<br />
<u>**Example**</u>

```lua
local ox_inventory = exports.ox_inventory

local water = ox_inventory:Search(source, 1, 'water')
for k, v in pairs(water) do
    print('\n______________'..'\n- index '..k)
    print(v.name, 'slot: '..v.slot, 'metadata: '..json.encode(v.metadata))
    water = v
    break
end

water.metadata.type = 'clean'
ox_inventory:SetMetadata(source, water.slot, water.metadata)
print(('スロット %s の %sx 個の水を新しいメタデータで変更しました'):format(water.slot, water.count))
```
