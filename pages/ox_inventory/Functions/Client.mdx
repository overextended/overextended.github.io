# クライアント

## openInventory

渡されたデータを使用してインベントリを開きます。

```lua
exports.ox_inventory:openInventory(invType, data)
```

- invType: `string`
  - `'player'`
  - `'shop'`
  - `'stash'`
  - `'crafting'`
  - `'container'`
  - `'drop'`
  - `'glovebox'`
  - `'trunk'`
  - `'dumpster'`
- data: `number` or `string` or `table`

<br />
<u>**Examples**</u>

<Tabs items={['Player', 'Shop', 'Stash']}>
  <Tab>
    ターゲットプレイヤーのインベントリを開きます。

    ```lua
    exports.ox_inventory:openInventory('player', 3)
    ```
  </Tab>
  <Tab>
    4番目の「雑貨店」の[場所](https://github.com/overextended/ox_inventory/blob/main/data/shops.lua#L14)を開きます。

    ```lua
    exports.ox_inventory:openInventory('shop', { type = 'General', id = 4 })
    ```
  </Tab>
  <Tab>
    [data/stashes](https://github.com/overextended/ox_inventory/blob/main/data/stashes.lua#L2-L19)の最初のスタッシュを開きます。

    ```lua
    exports.ox_inventory:openInventory('stash', 1)
    ```

    カスタムスタッシュを開きます（サーバーでRegisterStashを使用して作成）。

    ```lua
    exports.ox_inventory:openInventory('stash', 'society_police')
    ```

    特定の所有者のスタッシュを開きます。

    ```lua
    exports.ox_inventory:openInventory('stash', { id = 'police_locker', owner = 'license:xxxxxxxx' })
    ```
  </Tab>
</Tabs>

## openNearbyInventory

可能であれば、近くのプレイヤーのインベントリを開きます。

インベントリを開こうとするプレイヤーは自分のインベントリを開くことができなければならず、
プレイヤーが警察の仕事をしていない場合、ターゲットプレイヤーは致命傷を負っているか、
死亡アニメーションのいずれかを再生している必要があります。

```lua
exports.ox_inventory:openNearbyInventory()
```

## Items

登録されているすべてのアイテムのテーブルを返します。形式は[data/items.lua](https://github.com/overextended/ox_inventory/blob/main/data/items.lua)で定義されているとおりです。

オプションでアイテムの名前を取り、そのアイテムのデータのみを返します（すべてのデータを取得することはお勧めしません）。

```lua
exports.ox_inventory:Items(itemName)
```

- itemName?: `string`

次のスニペットは、サーバーから情報を取得するのではなく、okokCraftingやcore_craftingなどのクラフトリソースで使用できます。

```lua
local itemNames = {}

for item, data in pairs(exports.ox_inventory:Items()) do
    itemNames[item] = data.label
end
```

## useItem

渡されたアイテムを使用し、次にコールバック関数をトリガーします。
組み込みメソッド（サーバーチェック、プログレスバーなど）を利用するには、アイテムコールバック中に呼び出す必要があります。

```lua
exports.ox_inventory:useItem(data, cb)
```

- data: `table`
- cb?: `function`

```lua
exports('bandage', function(data, slot)
    local playerPed = PlayerPedId()
    local maxHealth = GetEntityMaxHealth(playerPed)
    local health = GetEntityHealth(playerPed)

    -- 歩行者は回復する必要がありますか？
    if health < maxHealth then
        -- 包帯を使用する
        exports.ox_inventory:useItem(data, function(data)
            -- アイテムが使用されたので、効果をトリガーします
            if data then
                SetEntityHealth(playerPed, math.min(maxHealth, math.floor(health + maxHealth / 16)))
                lib.notify({description = 'もう気分が良くなりました'})
            end
        end)
    else
        -- アイテムを使用しないでください
        lib.notify({type = 'error', description = '今は包帯は必要ありません'})
    end
end)
```

## setStashTarget

セカンダリインベントリキーを強制的に渡されたインベントリを開くようにします。マーカー内に立っている間にインベントリアクセスを有効にするのに役立ちます。

```lua
exports.ox_inventory:setStashTarget(id, owner)
```

- id: `string` or `number`
  - スタッシュID。
- owner?: `string` or `number`

<br />
<u>**Example**</u>

```lua
exports.ox_inventory:setStashTarget('motel5', 'bobsmith')
```

## getCurrentWeapon

現在装備されている武器のデータを取得します。

```lua
exports.ox_inventory:getCurrentWeapon()
```

イベントハンドラーを使用して、現在の武器の変更をリッスンすることもできます。

```lua
AddEventHandler('ox_inventory:currentWeapon', function(currentWeapon)
	CurrentWeapon = currentWeapon
end)
```

- currentWeapon?: `table`
  - ammo?: `string` 弾薬として使用されるアイテムの名前。
  - hash: `number`
  - label: `string`
  - melee: `boolean`
  - metadata: `table`
    - ammo?: `number` 武器に装填されている弾薬の量。
    - components?: `table` 武器コンポーネントを適用するために使用されるコンポーネントアイテム名の配列。
    - durability?: `number`
    - registered?: `string` ショップで武器を購入したプレイヤーの名前。
    - serial?: `string`
  - name: `string` アイテムの名前。
  - slot: `number`
  - weight: `number`

## displayMetadata

ツールチップに表示するメタデータプロパティを設定します。

```lua
exports.ox_inventory:displayMetadata(metadata, value)
```

- metadata: `string` または `table<string, string>` または `{ [string], [string] }`
  - metadataが文字列の場合、表示したいメタデータプロパティであり、その場合valueはオプションでは**ありません**。
  - キーと値のペアのテーブルにすることができます。キーはメタデータプロパティ、値はそのプロパティのラベルです。
  - 文字列配列の配列にすることができます。つまり、表示順序を設定するには `{ {'key', 'label' }, {'key2', 'label2' }` のようにします。
- value?: `string`
  - 表示される文字列メタデータプロパティのラベル。

<br/>
<u>**Example**</u>

```lua
exports.ox_inventory:displayMetadata('mustard', 'Mustard')
```

```lua
exports.ox_inventory:displayMetadata({
    mustard = 'Mustard',
    ketchup = 'Ketchup'
})
```

## giveItemToTarget

プレイヤーのインベントリから別のプレイヤーにアイテムを与えます。

```lua
exports.ox_inventory:giveItemToTarget(serverId, slotId, count)
```

- serverId: `number`
  - ターゲットプレイヤーのserverId。
- slotId: `number`
  - 与えるアイテムのslotId。
- count?: `number`
  - 与えるアイテムの量。`nil`、`0`、またはスロット数を超える値を指定すると、スタック全体が渡されます。
## weaponWheel

武器ホイールを有効にしますが、インベントリ武器の使用を無効にします。

主に武器化された車両に使用されますが、「ミニゲーム」用に呼び出すこともできます。

```lua
local exports.ox_inventory:weaponWheel(state)
```

- state: `boolean`

## Search

インベントリでアイテムまたはアイテムのリストを検索します。結果は最初の引数によって異なります。

```lua
exports.ox_inventory:Search(search, item, metadata)
```

- search: `'slots'` または `'count'`
  - `'slots'` は、アイテムが見つかったスロットのテーブルを返します。
  - `'count'` は、プレイヤーのインベントリ内の指定されたアイテムの数を返します。複数のアイテムを検索する場合、
    itemName = count のキーと値のペアを返します。
- item: `table` または `string`
  - 単一のアイテム名またはアイテム名の配列を指定できます。
- metadata?: `table` または `string`
  - metadataが文字列として提供されている場合、アイテムの `metadata.type` プロパティを検索します。

### Count

## Statebags

### invBusy

プレイヤーのインベントリが現在アクションを実行しているかどうか（つまり、アイテムを使用しているかどうか）を返します。
trueに設定すると、インベントリを開くことを無効にできます。

- invBusy: `boolean`

```lua
local invBusy = LocalPlayer.state.invBusy

if invBusy then
    -- ビジー状態のときに何かをする
else
    -- ビジー状態でないときに何かをする
end
```

#### インベントリを開くのを無効にする

```lua
LocalPlayer.state.invBusy = true
```

### invHotkeys

プレイヤーのインベントリホットキーへのアクセスを有効/無効にできます。

- invHotkeys: `boolean`

```lua
LocalPlayer.state.invHotkeys = false
```

### invOpen

プレイヤーのインベントリが現在開いているかどうかを返します。

- invOpen: `boolean`

```lua
local invOpen = LocalPlayer.state.invOpen

if invOpen then
    -- 開いているときに何かをする
else
    -- 閉じているときに何かをする
end
```

### canUseWeapons

プレイヤーの武器の使用を有効/無効にできます。

```lua
LocalPlayer.state.canUseWeapons = false
```
