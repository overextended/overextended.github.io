# フック

イベントフックを使用すると、サードパーティのリソースはインベントリコードを直接変更せずに新しい動作を定義できます。

## registerHook

```lua
exports.ox_inventory:registerHook(eventName, function(payload) end, options)
```

- eventName: `string`
- payload: `table`
- options?: `table`
  - print?: `boolean`
    - イベントをトリガーするときにコンソールに出力します。
  - itemFilter?: `{ [string]: true }`
    - イベントは、セット内でキーとして定義されたアイテムに対してのみトリガーされます。
  - inventoryFilter?: `string[]`
    - イベントは、配列内の[パターン](<http://www.easyuo.com/openeuo/wiki/index.php/Lua_Patterns_and_Captures_(Regular_Expressions)>)のいずれかに一致するインベントリに対してのみトリガーされます。
  - typeFilter?: `{ [string]: true }`
    - イベントは、提供されたタイプのいずれかを持つインベントリ（例：「player」、「stash」）に対してのみトリガーされます。

戻り値:

- hookId: `number`

### swapItems

あるスロットから別のスロットにアイテムを移動するとき、またはアイテムを「与える」ときにトリガーされます。
`false` を返すことで、アクションをキャンセルし、インベントリの状態を元に戻すことができます。

- Payload: `table`
  - source: `number`
  - action: `'move'` または `'stack'` または `'swap'` または `'give'`
  - fromInventory: `table` または `string` または `number`
  - toInventory: `table` または `string` または `number`
  - fromType: `string`
  - toType: `string`
  - fromSlot: `table`
  - toSlot?: `table` または `number`
  - count: `number`

**例**

「water」がグローブボックスやトランクに移動されたり、そこから移動されたりするのをブラックリストに登録します。

```lua
local hookId = exports.ox_inventory:registerHook('swapItems', function(payload)
    print(json.encode(payload, { indent = true }))
    return false
end, {
    print = true,
    itemFilter = {
        water = true,
    },
    inventoryFilter = {
        '^glove[%w]+',
        '^trunk[%w]+',
    }
})
```

### openInventory

- Payload: `table`
  - source: `number`
  - inventoryId: `number` または `string`
  - inventoryType: `string`

プレイヤーがセカンダリインベントリを開こうとするとトリガーされます。
`false` を返すことで、アクションをキャンセルし、プレイヤーのインベントリを閉じたままにすることができます。

**例**

グローブボックスとトランクを無効にします。

```lua
local hookId = exports.ox_inventory:registerHook('openInventory', function(payload)
    print(json.encode(payload, { indent = true }))
    return false
end, {
    print = true,
    inventoryFilter = {
        '^glove[%w]+',
        '^trunk[%w]+',
    }
})
```

### createItem

- Payload: `table`
  - inventoryId?: `number` または `string`
  - metadata: `table`
  - item: `table`
  - count: `number`

アイテムが作成されたとき（購入、AddItemの使用、またはインベントリデータの変換時）にトリガーされます。
テーブルを返すことで、アイテムに与えられたメタデータを変更または置換できます。

**例**

「water」のラベルを「ミネラルウォーター」に設定します。

```lua
local hookId = exports.ox_inventory:registerHook('createItem', function(payload)
    print(json.encode(payload, { indent = true }))
    local metadata = payload.metadata
    metadata.label = 'Mineral Water'
    return metadata
end, {
    print = true,
    itemFilter = {
        water = true
    }
})
```

### buyItem

- Payload: `table`
  - source: `number`
  - shopType: `string`
  - shopId: `number`
  - toInventory: `number`
  - toSlot: `number`
  - itemName: `string`
  - metadata: `table`
  - count: `number`
  - price: `number`
  - totalPrice: `number`
  - currency?: `string`

アイテムが購入されようとしているときにトリガーされ、トランザクションを防ぐために `false` を返すことができます。

**例**

プレイヤーが雑貨店でアイテムを購入できないようにします。

```lua
local hookId = exports.ox_inventory:registerHook('buyItem', function(payload)
    print(json.encode(payload, { indent = true, sort_keys = true }))
    return false
end, {
    print = true,
    itemFilter = {
        water = true
	  },
})

```

### craftItem

- Payload: `table`
  - source: `number`
  - benchId: `number` | `string`
  - benchIndex: `number`
  - recipe: `table`
    - count: `number`
    - duration: `number`
    - ingredients: `table<string, number>`
    - name: `string`
    - slot: `number`
    - weight: `number`
  - toInventory: `number`
  - toSlot: `number`

**例**

プレイヤーがロックピックを作成できないようにします。

```lua
local hookId = exports.ox_inventory:registerHook('craftItem', function(payload)
    print(json.encode(payload, { indent = true, sort_keys = true }))
    return false
end, {
    print = true,
	itemFilter = {
		lockpick = true
	},
})

```

## removeHooks

指定されたIDを持つ呼び出し元リソースによって作成されたフックを削除します。
IDが指定されていない場合、リソースによって登録されているすべてのフックが削除されます。

```lua
exports.ox_inventory:removeHooks(id)
```

- id?: `number`
